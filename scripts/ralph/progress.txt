## Codebase Patterns
- Root package.json is `claude-ralph`, type: module, bin: `./dist/bin/ralph.js`
- Root tsconfig.json compiles src/ -> dist/, target ES2020, module ES2020
- Dashboard is a Next.js app in dashboard/ with its own package.json
- scripts/ralph/ contains runtime data (prd.json, progress.txt, archive/); scripts/ contains shared tools (ralph.sh, CLAUDE.md)
- Use `npx tsc --noEmit` for root TypeScript checks; `cd dashboard && npm run build` for dashboard checks
- CLI uses Commander.js with ESM imports; version read from package.json via readFileSync (not import assertion)
- `npm link` at root makes `ralph` command available globally for testing
- Global config at ~/.ralph/config.json; use src/lib/global-config.ts for read/write; atomic writes use tmp file in same dir (not os.tmpdir) to avoid cross-drive rename failures on Windows
- Dashboard uses Next.js 16 with tsx for WS server; `next start` for production, standalone mode when available
- On Windows, use `.cmd` suffixed binaries from node_modules/.bin (next.cmd, tsx.cmd) and set shell: true for spawn
- Port check: use net.createServer trick to test if port is available before starting servers
- Dashboard config (dashboard/src/lib/config.ts) mirrors src/lib/global-config.ts structure; all API routes use getActiveProjectPaths() for project-relative paths
- ws.ts (WebSocket server) runs standalone via tsx and imports from relative paths (../src/lib/), not @/lib/ aliases
- Next.js 16 dynamic route params are `Promise<{ name: string }>` — must be awaited
- Pre-existing lint errors (7 total) won't block commits; they're in ws.ts, logs, story components, websocket hook
- Next.js standalone output: server.js may be at .next/standalone/server.js or .next/standalone/dashboard/server.js depending on workspace root detection
- npm publish: files field is whitelist; need dashboard/.npmignore to override dashboard/.gitignore's .next/ exclusion
