## Codebase Patterns
- PID file path: scripts/ralph/.ralph-pid (SCRIPT_DIR-relative)
- `kill -0 $pid` works on both Unix and Windows (Git Bash) for process liveness check
- ralph.sh uses `set -e` + `trap cleanup EXIT` for cleanup on any exit
- ralph.sh reads config from ~/.ralph/config.json with CLI args as override
- Skills are markdown files at skills/<name>/SKILL.md with frontmatter (name, description, user-invocable)
- ralph.sh CLI: `--tool amp|claude --timeout <min> --webhook <url> <max_iterations>`
- /ralph-stop uses platform detection (uname -s) to choose between taskkill (Windows) and kill (Unix)
- On Windows (Git Bash), taskkill uses //pid and //t //f (double-slash for MSYS path translation)
- `getPidFilePath()` in ralph-process.ts resolves PID path from active project config, reusable across modules
- chokidar needs `add`/`unlink` handlers for file creation/deletion, not just `change`
- `request.json()` throws on empty body — wrap in try/catch for optional-body API routes
- ralph.sh CLI arg order: `--tool X [--timeout Y] [--webhook Z] <maxIterations>` (positional last)
