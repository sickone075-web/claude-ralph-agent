## Codebase Patterns
- Dashboard is a Vite SPA with pages in `dashboard/src/pages/` and components in `dashboard/src/components/`
- Router is in `dashboard/src/router.tsx` using react-router-dom v7
- Pre-existing TS errors in `server/routes/archives.ts` and `server/routes/projects.ts` (string | string[] type issues) — not introduced by this work
- Sidebar nav items are defined in `dashboard/src/components/sidebar.tsx` as a `navItems` array
- Project uses Tailwind v4 with CSS variables in `dashboard/src/globals.css`
- Claude Light Theme: all CSS variables are hex-based (not oklch), defined in `:root` and `.dark` blocks in globals.css
- Font stack: Inter (UI), JetBrains Mono (code/data), Cormorant Garamond (headings/serif) — imported via Google Fonts in index.html
- `--font-serif` CSS variable available for Cormorant Garamond via `var(--font-serif)` or `font-serif` Tailwind class
- WebSocket provider wraps dashboard layout in router.tsx
- Dashboard source is at `dashboard/` from repo root (NOT under `scripts/ralph/dashboard/`)
- Design system reference is at `design-system/MASTER.md` from repo root
- RalphStatusIndicator supports `compact` prop for icon-only display
- Server log cache module is at `dashboard/server/log-cache.ts` — tracks current Story ID and caches logs by story
- Express v5 `req.params` returns `string | string[]` — use `as string` cast for single params
- WebSocket `ralph:output` events now include `storyId` field (string | null)
- Git-specific API routes are under `/api/git/*` in `dashboard/server/routes/git.ts`
- WebSocket context: `useWsSubscribe()` from `websocket-provider.tsx` lets any child component subscribe to WS events without creating new connections
