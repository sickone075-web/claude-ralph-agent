# Ralph Progress Log
Started: 2026-03-02

## Codebase Patterns
- Next.js 16 App Router with Tailwind CSS v4 + shadcn/ui in `dashboard/` directory
- Dark theme enforced via `className="dark"` on `<html>` element
- Sidebar layout uses `(dashboard)` route group with shared layout.tsx
- shadcn/ui components installed via `npx shadcn@latest add <component>`
- Sidebar colors: zinc-950 background, zinc-800 selected item, zinc-400 text
- All pages are at `app/(dashboard)/*/page.tsx`
- Root page (`/`) redirects to `/dashboard`
- API routes go in `app/api/` directory using Next.js Route Handlers
- API responses use `{ data: T | null, error: string | null }` format
- Config helper in `lib/config.ts` reads optional `dashboard/config.json` with defaults
- Shared types in `lib/types.ts` (PRD, Story, ApiResponse)
- PRD file read/write helper in `lib/prd-file.ts` with atomic writes and in-memory mutex
- Write operations: write to .tmp file then rename for atomic updates
- Progress log parser in `lib/progress-parser.ts` splits by `---` separator and extracts structured records
- simple-git library for git history access; project root is `path.resolve(process.cwd(), "..")`
- Ralph process manager singleton in `lib/ralph-process.ts` with event listener pattern for WebSocket integration
- Process control API routes under `app/api/ralph/` (start, stop, stdin, status)
- On Windows, use `bash` to spawn `.sh` scripts; on Unix, spawn script directly
- Process termination: Windows uses `taskkill /pid /t /f`, Unix uses SIGTERM
- WebSocket server in `server/ws.ts` runs standalone on port 3001 via `tsx`, imports ralph-process for event piping
- `npm run dev` uses `concurrently` to run Next.js + WebSocket server together
- chokidar watches prd.json and progress.txt for file change events broadcast via WebSocket
