## Codebase Patterns
- Story type is in `dashboard/src/lib/types.ts`, used by all API routes and components
- PRD read/write uses atomic file operations (tmp + rename) in `dashboard/src/lib/prd-file.ts`
- API routes at `dashboard/src/app/api/prd/stories/[id]/route.ts` handle story CRUD
- WebSocket server at `dashboard/server/ws.ts` broadcasts file change events
- Dashboard uses zustand store at `dashboard/src/lib/store.ts` for state management
- Dark theme enforced globally: zinc-900 bg, zinc-800 borders, cyan/blue accents
- Story status determined by: `passes=true` → completed, `id === currentStoryId && ralphRunning` → running, else pending
- shadcn/ui components in `dashboard/src/components/ui/`
- Flow components (ReactFlow-based) live in `dashboard/src/components/flow/`
- @xyflow/react styles imported globally in globals.css; ReactFlow components need `"use client"`
- ReactFlow custom nodes: wrap with `memo()`, export data interface, use `Handle` for connections
- TypeScript check: `cd dashboard && npx tsc --noEmit`
- ReactFlow Node data needs `as unknown as Record<string, unknown>` cast for custom data interfaces
- `nodeTypes` object must be defined outside component to prevent ReactFlow re-registration warnings
