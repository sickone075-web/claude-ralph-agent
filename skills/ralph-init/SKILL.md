# 首次初始化项目 AI 上下文

首次扫描项目，生成为 Ralph 无记忆 agent 优化的根级与模块级 CLAUDE.md。

## 使用方法

```bash
/ralph:init <项目摘要或名称>
```

> 此命令仅用于**首次生成**。已有 CLAUDE.md 后需要更新，请使用 `/ralph:update`。

## 核心理念

Ralph 系统每次迭代启动一个**全新的、无记忆的 Claude Code 实例**。根级和模块级 CLAUDE.md 被 Claude Code **自动加载**，是最高效的上下文注入通道。因此生成的文档必须让 agent：
- **第一时间知道**能用什么命令验证代码
- **立即理解**项目技术选型和结构
- **快速定位**做某类任务该改哪些文件

## 你的角色

你是**协调者**，负责调用子智能体完成项目扫描与文档生成。

---

## 执行工作流

**必须按以下步骤执行，使用 Agent 工具调用子智能体**

### 步骤 1：获取当前时间戳

**必须首先调用 `get-current-datetime` 子智能体**：

```
Agent({
  subagent_type: "get-current-datetime",
  prompt: "获取当前日期时间，用于文档时间戳",
  description: "获取当前时间"
})
```

等待返回时间戳后，保存为 `$TIMESTAMP` 供后续使用。

### 步骤 2：检测 Ralph 环境就绪状态

在调用子智能体前，**协调者自行检查**以下文件/目录是否存在（使用 Glob 工具）：
- `~/.ralph/` 目录（全局）
- `~/.ralph/ralph.sh`（全局脚本）
- `.ralph/prd.json`（项目级）
- `~/.ralph/config.json`

记录缺失项，在步骤 4 的摘要中提示用户。

### 步骤 3：调用初始化架构师

**使用 `init-architect` 子智能体执行完整扫描**：

```
Agent({
  subagent_type: "init-architect",
  prompt: `扫描项目并生成 Ralph 迭代优化的 CLAUDE.md 文档。

项目摘要：<用户提供的摘要>
当前时间戳：$TIMESTAMP
工作目录：$PWD

## 核心目标

生成的 CLAUDE.md 是给「无记忆的 Claude Code 实例」看的。这个实例需要在一个上下文窗口内理解项目并实现一个用户故事。因此文档必须：
1. 把最常用的信息放在最前面
2. 操作指南要具体到文件路径
3. 避免冗余的泛泛描述

## 根级 CLAUDE.md 必须包含以下章节（按此顺序）

### 1. 快速参考（Quick Reference）
扫描 package.json / Makefile / Cargo.toml 等，提取实际可用的命令：
- 构建命令
- 类型检查命令
- 测试命令（含单元测试和集成测试）
- Lint / Format 命令
- 开发服务器启动命令（含端口号）
- 其他常用脚本

### 2. 技术栈
列出项目实际使用的：
- 框架及版本
- 语言及版本
- 包管理器
- 数据库（如有）
- 测试框架
- 其他关键依赖

### 3. 项目结构
- 生成 Mermaid 图展示模块关系
- 每个顶层目录的一行说明

### 4. 关键约定
扫描代码推断项目的：
- 路由模式 / API 风格
- 状态管理方式
- 组件 / 模块命名规范
- 导入路径别名（如 @/ 等）
- 错误处理模式
- 日志规范

### 5. 常见操作指南
根据项目类型，生成具体到文件路径的操作指南：
- 添加 API 端点：修改哪些文件 → 更新哪些类型 → 添加哪些测试
- 添加 UI 页面 / 组件：创建文件位置 → 路由注册 → 样式文件
- 添加数据库字段 / 表：迁移文件 → ORM 模型 → 类型定义
- 添加测试：测试文件位置 → 命名规范 → mock 模式

## 模块级 CLAUDE.md 要求

每个识别到的模块目录生成 CLAUDE.md，必须回答：
1. **职责**：这个模块做什么？（一句话）
2. **入口文件**：主要入口在哪？
3. **联动文件**：改这里通常还需要改哪里？
4. **代码模式**：这个目录遵循的模式 / 风格
5. **已知注意事项**：有哪些坑或特殊处理？

## 执行阶段

1. 阶段 A：全仓清点（文件统计、模块识别）
2. 阶段 B：模块优先扫描（入口、接口、依赖、测试）
3. 阶段 C：深度补捞（按需）
4. 阶段 D：生成文档（根级 + 模块级 CLAUDE.md，严格按上述章节结构）

输出覆盖率报告与推荐下一步。`,
  description: "初始化项目文档"
})
```

## 安全边界

1. **只读/写文档** — 不改源代码
2. **忽略生成物** — 跳过 `node_modules`、`dist`、二进制文件
3. **环境检测只提醒** — 不自动创建 Ralph 配置文件

## 关键规则

1. **必须使用 Agent 工具**调用子智能体，不要自己执行扫描逻辑
2. 先调用 `get-current-datetime` 获取时间戳
3. **协调者自行**检查 Ralph 环境就绪状态（步骤 2）
4. 再调用 `init-architect` 执行完整扫描
5. 结果在主对话打印摘要，全文由子智能体写入仓库
6. 摘要必须包含 Ralph 环境就绪状态
7. 如果根级 CLAUDE.md 已存在，提醒用户改用 `/ralph:update`
